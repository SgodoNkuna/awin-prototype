## Risks & Gotchas

- **OAuth redirect URI:** Make sure Supabase Google OAuth app has `http://localhost:8080/auth/callback` (or your production domain) registered as a valid redirect URI. This is configured in Supabase Dashboard → Authentication → Providers → Google.
- **Environment variables:** The current setup expects `VITE_SUPABASE_URL` and `VITE_SUPABASE_PUBLISHABLE_KEY` in `.env`. Verify these are set.
- **Lovable auto-generation:** Some files (like `src/integrations/supabase/client.ts` and `auth-middleware.ts`) have "auto-generated by Lovable" comments. These are safe to edit; they're not re-generated once you remove the Lovable tooling.
- **TypeScript plugins:** Make sure `@vitejs/plugin-react` is installed (it should be). If not, add it: `bun add -d @vitejs/plugin-react`.
